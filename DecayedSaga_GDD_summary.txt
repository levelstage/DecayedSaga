1. 핵심 컨셉
================================================================
- 강령술사(인형사)가 진흙인형에 원혼이 깃든 유품을 장착해 싸움
- 인형사는 전장에 직접 등장하지 않음. 숨어서 인형 조종
- 뱅(BANG!) 방식 카드 전투 + 파이어엠블렘식 이동 + 브레이크 시스템
- 퍼마데스: HP 0 = 인형 영구 소실
- 단일 엔딩 (리치 봉인 = 클리어)
- 비주얼: 보드게임 미니어처 풍. 정적인 2D 입체 그림. 애니메이션 없음
- 세계관 전달: 프롬 소프트웨어식. 유품 플레이버 텍스트/이벤트로 암시
- 맵 에디터: Tiled Map Editor (JSON 내보내기)

2. 스토리 & 세계관
================================================================

[배경]
한때 번성했던 마을. 흑마법사가 현자로 위장해 영주를 암살하고
언데드 군세로 지배. 10년 전 탈환을 위한 토벌군이 파견됐으나
리치의 눈을 보는 순간 대량 언데드화. 살아남은 자들은 눈을 감거나
가리거나 찌르며 필사적으로 싸워 시간을 벌었고, 그 사이 신관들이
봉인의 안개를 설치. 리치는 봉인됐지만 토벌은 실패.
봉인의 안개는 지금도 신관들에 의해 유지중.

[리치의 능력]
- 핵심: 자신의 눈을 본 대상의 영혼을 추출 → 언데드화
- 시선 기반 능력. 살아있는 인간에게만 작동
- 직접 접촉 시 유품에 깃든 원혼도 추출 가능
- 영혼 없는 진흙 인형은 추출 불가

[유품과 원혼]
- 리치에게 영혼을 뽑혀 죽은 자 중 강한 원한을 품은 자들이
  저세상으로 가지 못하고 물건에 깃듦
- 리치가 직접 접촉하면 추출 가능하나 인형을 통해 싸우는
  주인공에게는 접근하기 어려움
- 마을 해방을 원하는 원혼들이 자발적으로 유품을 통해 함께 싸움

[주인공]
- 마을 출신. 영주의 아들 (최종 던전 성의 수기로만 밝혀짐)
- 신관을 설득해 봉인의 안개를 뚫고 입장
- 안대 착용: 리치의 눈을 보지 않기 위한 필수 장비
- 인형 사용 이유: 인형은 눈이 감지 기관이 아님 → 리치 능력 면역
- 전투 시 숨어서 인형 조종. 적들: "누구냐" "정체를 밝혀라" 외침
- 리치도 끝까지 정체를 모름. 플레이어만 수기로 알게 됨
- 외형: 검은 로브를 어깨에 걸침(입은 게 아님). 안에 붉은 상의.
        검은 안대. 갈색 머리 (중세 판타지 버전 타츠미 코타로 패러디)

[엔딩]
리치와 전투 → 승리 → 리치 영혼을 인형에 봉인
→ 마을 전체 원혼 해방 → 주인공만 홀로 남음. 끝.

[세계관 전달 수단]
- 유품 플레이버 텍스트: 실패한 도전자들의 흔적. 원혼들의 이야기
- 성의 주인공 방 수기: 영주의 아들임을 플레이어만 알게 됨
- 적 대사: "정체를 밝혀라" "누구냐" — 아무도 모름
- 이벤트 텍스트: 마을 곳곳의 흔적

3. 행동 게이지 (CTB)
================================================================
- 게이지 만충 = 4 (AG 1.0 = 1턴)
- 모든 유닛 감소 속도 동일
- GaugeWeight 기준:
  AG 0.25=1 / AG 0.5=2 / AG 0.75=3 / AG 1.0=4 / AG 1.5=6 / AG 2.0=8

================================================================
4. 카드 시스템
================================================================

[공개카드 — 순수 패시브]
- 장비당 1장. 인형당 최대 3장 (무기/장갑/소품)
- 행동권 시 직접 사용 없음. 스탯과 토큰만 명시
- 무기: 제압 N, 속성 명시 + 토큰
- 장갑: 이동력 N, 약점 속성 명시 + 토큰
- 소품: 스탯 없음 + 토큰만
- 토큰: 장착 시 자동 부착. 브레이크 시 소멸. 복구 시 초기 토큰 재부착

[토큰 종류]
수치형:
- 내구도: 0이 되면 브레이크 (MVP)
- 강화: 수만큼 공격력 +1. 해당 슬롯 스킬에만 적용 (MVP)
- 방호: 수만큼 피해 -1. 해당 슬롯 스킬에만 적용 (MVP)
  예) 장갑 방호1 → 방어 카드 경감에만 적용. 무기 막기에는 미적용
- 부식/맹독/마비: 추후 추가

트리거형:
- 공개카드 아래 슬롯에 장착. 액티브 카드 사용으로 장착
- 피격 트리거 외 다양한 트리거 발동 가능
- 제출 시 소모
- 예: 반격 토큰 (장갑 카드로 무기 슬롯에 장착)
  → 피격 트리거: 공격자가 무기 사거리 안이면 피해 N

[액티브 스킬 카드]
- 행동권 시 핸드에서 1장 제출
- GaugeWeight만큼 게이지 재충전
- 사거리 미명시 시 무기 공개카드 사거리 참조
- 슬롯 브레이크 시 사용 불가

[트리거 스킬 카드]
- 조건 충족 시 핸드에서 제출. 행게 증가 없음
- 원하는 만큼 여러 장 제출 가능
- 피격 트리거 타이밍: 적 공격 선언 후 피해 계산 직전
- 드로우: 트리거 카드를 제출한 턴에 몇 장을 냈든 1장만 드로우
- 슬롯 브레이크 시 사용 불가

[방어/막기 카드]
방어 (장갑 슬롯):
- 피격 트리거: 피해 N 경감
- 사용시: 이동력만큼 이동. AG 1.0

막기 (무기 슬롯):
- 피격 트리거: 피해 N 경감
- 사용시: 이동력만큼 이동. AG 0.5 (이동이 더 쌈)

[행동권 흐름]
1. 행동권 획득 (게이지 0)
2. 드로우 (GaugeWeight/4 누적. 1 이상 시 1드로우)
3. 행동 선택:
   ① 액티브 카드 1장 제출 → 효과 발동 → GaugeWeight만큼 재충전
   ② 패스 → 드로우 1장 → AG 1.0 재충전
4. 핸드 정리: 4장 이상이면 1장 버리기
※ 트리거: 조건 충족 시 언제든 제출 가능. 해당 턴 드로우는 1장만

5. 공격 속성 & 제압
================================================================
참격: 경장갑 취약. 딜 높음. 날붙이류
충격: 중장갑 취약. 브레이크 특화. 둔기류
관통: 약점 적 희귀. 딜 높거나 사거리 김
특수: 약점 적 없음. 물리 이외 속성

※ 모든 공격은 피해 0이어도 공격으로 취급 — 반격 등 피격 트리거 발동

제압: 무기 공개카드에 명시. 공격 시 대상 장갑 내구도 감소
취약 속성 공격 시 제압 2배
카드 제압+N 키워드: 해당 카드 사용 시 무기 제압에 추가

6. 이동
================================================================
- 파이어엠블렘식 N칸 자유 이동
- 이동 수단: 방어(AG 1.0) / 막기(AG 0.5) 카드 사용시 효과
- 이동력: 장갑 공개카드에 명시
- 장갑 브레이크 시 방어 카드 사용 불가 → 이동 불가

================================================================
7. 브레이크
================================================================
- 내구도 0 → 공개카드 뒤집힘 + 해당 슬롯 카드 사용 불가 + 토큰 소멸
- 자동 회복: 장비별 명시된 턴 후 자동 복구. 초기 토큰도 함께 복구
- 액티브 수복: 특수 스킬 있는 경우만 가능
- 데미지: 피해 + 강화 토큰 - 방호 토큰 - 방어 카드 경감 (최소값 없음. 0뎀 가능)

10. 유품 시스템
================================================================
- 유니크 장비. 무기/장갑/소품 중 하나의 슬롯 차지
- 일반 장비보다 토큰 풍부 + 특수 스킬 보유 가능
- 인형당 1개 제한. 장착 시 해당 영혼의 외형으로 인형 변화
- 귀속 유품: 분리 시 진흙 비용 발생. 퍼마데스 시 회수 가능

================================================================
11. 소품 계열
================================================================
- 방어구형: 방어 카드 추가 → 이동/방어 강화
- 무기형: 공격 카드 추가 → 화력 강화. 이동/방어 희생
- 신발형: 이동력 증가 or 방어 카드 절대량 증가
- 장신구형: 특이한 효과 (드로우 조작, 토큰 조작 등)

================================================================
12. 슬롯 레벨 & 성장
================================================================
- 인형당 슬롯별 독립 레벨 (WeaponSlotLevel / ArmorSlotLevel / AccessorySlotLevel)
- 대표 레벨 = max(3슬롯). HP 기준
- 슬롯 레벨 = 해당 슬롯 장비 티어 상한
- 진흙 재화: 인형 생성 / 슬롯 레벨업 / 귀속 유품 분리 / 장비 구매
- 난이도: 이지/노말/하드 = 진흙 수급량만 다름
- 티어 네이밍: 티어1 농부의 / 티어2 도적의 / 티어3 용병의 / 티어4 기사의 / 티어5 영웅의

================================================================
13. 전투 시스템
================================================================
- 그리드 기반. 초반 5×5 → 중반 6×7 → 후반 8×8
- 투입 인형 최대 3마리. 6인 엔트리 (전투 3 + 대기 3)
- 전투 중 교체 불가
- 최소 피해 없음. 0뎀 가능
- 적 AI: 결정론적. 카드 우선순위 순서대로 CanUse 판단

================================================================
14. 게임 시작
================================================================
지급 유품: 농부의 장검 / 농부의 나무 갑옷 / 땜장이의 두건

시작 인형:
- 인형 A: 농부의 장검 + 진흙 피부 → 딜러
- 인형 B: 주먹 + 농부의 나무 갑옷 → 탱커
- 인형 C: 주먹 + 진흙 피부 + 땜장이의 두건 → 서포터

5×5 맵에서 시작.

15. 적 목록 — 거주 구역 (HP 공통 5)
================================================================

레서 좀비
- 무기: 발톱 (제압1, 참격, 내구도2, 회복 1.0턴)
  덱: 할퀴기(피해2, 사거리2, AG 1.0)×2 / 물기(피해3, 사거리1, 제압+1, AG 1.0)×1
- 장갑: 썩은 피부 (이동력2, 참격취약, 내구도3, 회복 1.0턴)
  덱: 방어1×3

16. 프로젝트 구조
================================================================
- GameCore: C# Class Library (로직. MonoGame 의존성 없음)
- GameApp: MonoGame Project (렌더링/입력/사운드)

핵심 모델:
- Golem: Id, Name, WeaponSlotLevel, ArmorSlotLevel, AccessorySlotLevel,
         Hp, MaxHp, WeaponSlot, ArmorSlot, AccessorySlot,
         ActionGauge, AccumulatedTurns
- Equipment: Id, Name, SlotType, Tier, PublicCard, DeckCards[], BreakRecoveryTurns
- Relic: Equipment 상속 + IsBound, GolemSprite
- PublicCard: SlotType, Stat(제압/이동력/없음), Attribute(참격/충격/관통/특수),
              Weakness(취약속성), Tokens[], BreakRecoveryTurns
- DeckCard: CardId, Name, SlotType, CardType(Active/Trigger),
            TriggerCondition?, GaugeWeight, SkillClass, Description
- TriggerToken: TokenType, TriggerCondition, SkillClass (제출 시 소모)
- BattleContext: Caster, Targets[]
- IActiveSkill: CanUse(BattleContext), Execute(BattleContext)
- ITriggerSkill: CanTrigger(BattleContext), Execute(BattleContext)
- RangeData: type(Linear/Diagonal/All/Self), range(int)

데이터: JSON + C# 리플렉션
세이브: JSON
