================================================================
DECAYED SAGA — 게임 설계 문서 요약 (Claude Code 전달용)
================================================================
버전: v0.8
장르: 턴제 전략 (CTB 방식)
기술: C# + MonoGame (GameCore 로직 / GameApp 렌더링 분리)
맵 에디터: Tiled Map Editor (JSON 내보내기 → GameCore 파싱)

----------------------------------------------------------------
1. 핵심 컨셉
----------------------------------------------------------------
- 강령술사(인형사)가 진흙인형에 죽은 자의 영혼이 깃든 유품을 장착해 싸움
- 인형사는 전장에 등장하지 않음. 후방 지휘 개념
- 체스 기물 이동 + 카드(변주) 시스템 + 브레이크 시스템
- 퍼마데스: HP 0 = 인형 영구 소실
- 비주얼: 보드게임 미니어처 풍. 정적인 2D 입체 그림
- 세계관 연출: 프롬 소프트웨어식 — 유품 플레이버 텍스트/이벤트로 암시
- 단일 엔딩 (리치 처치 = 클리어)
- 주요 모티브: 리버스 1999, 글룸헤이븐, 체스, 슬레이더스파이어, 다크소울, 좀비랜드 사가, Into the Breach, Darkest Dungeon

----------------------------------------------------------------
2. 행동 게이지 (CTB)
----------------------------------------------------------------
- 게이지 만충 = 4 (= 1턴 기준)
- 모든 유닛 감소 속도 동일
- 행동 후 사용한 행동의 GaugeWeight만큼 재충전
- 공개카드 GaugeWeight 기준: 빠른=2 / 보통=4 / 느린=6 / 매우느린=8

----------------------------------------------------------------
3. 카드 & 변주 시스템
----------------------------------------------------------------
[공개카드 — 행동의 핵심]
- 장비당 1장. 인형당 최대 3장 (무기/장갑/소품 각 1장)
- 슬롯별 행동:
  * 무기 공개카드 → 공격 행동 (이동 옵션 포함 가능)
  * 장갑 공개카드 → 방어 행동 (방호 토큰 획득 + 턴 넘김, 이동 옵션 포함 가능)
  * 소품 공개카드 → 소품 고유 행동 (대부분 기본 효과 있음)
- 패시브: 공개카드에 토큰 형태로 부착 (내구도 토큰 등). 장착 시 자동 부착

[비공개 카드 — 변주 (Variation)]
- 행동의 성격을 바꾸거나 강화
- 공개카드 발동 시 핸드에서 추가로 버려서 적용 (선택적)
- 슬롯 귀속: 공격 변주→무기, 방어 변주→장갑, 소품 변주→소품. 예외 없음
- 독립 키워드: 해당 행동에 1장만 사용 가능
- 강력한 카드일수록 덱 매수 많음 (순환 견제)
- 3장짜리 무기/장갑은 카드 질이 낮게 설계 (3/3/0 빌드 견제)

[덱 구성]
- 무기 변주 3~7장 + 장갑 변주 3~7장 + 소품 변주 0~10장
- 전부 섞어서 하나의 덱으로 운용
- 드로우 시 슬롯 구분 없이 랜덤 등장

[행동권 흐름]
1. 행동권 획득 (게이지 0)
2. 드로우 (AccumulatedTurns 기반. GaugeWeight/4 누적. 1 이상 시 1드로우)
3. 행동 선택: ① 공개카드(공격/방어/소품) ② 기본 이동
4. 변주 선택 (선택적): 공개카드 시 핸드에서 버려서 강화/변형
5. 행동 발동
6. 핸드 정리: 핸드 3장 이상이면 1장 버리기 (플레이어 선택)
7. 게이지 재충전

----------------------------------------------------------------
4. 토큰 & 브레이크 시스템
----------------------------------------------------------------
[MVP 토큰 3종]
- 내구도: 모든 장비. 0이 되면 브레이크
- 강화: 무기 슬롯. 토큰 수만큼 공격력 +1
- 방호: 장갑 슬롯. 토큰 수만큼 피해 -1

[추후 추가]
- 부식: 최대 내구도 감소
- 맹독: 드로우 시 HP 피해. 발동마다 1감소
- 마비: 공격력 감소

[브레이크]
- 내구도 0 → 공개카드 뒤집힘 + 해당 슬롯 변주 사용 불가 + 토큰 전부 제거
- 유품 브레이크 추가: 기본 이동 무력화
- 전투 종료 시 자동 복구

[브레이크 수복]
- 행동권 1회 소모 + 같은 슬롯 변주 카드 N장 버리기
- N은 공개카드 뒷면에 명시. 장비마다 상이

[내구도 감소]
- 전용 스킬로만 감소 (일반 피해로는 줄어들지 않음)
- 슬롯 지정 방식: 무기 쳐내기, 장갑 쪼개기, 소품 파괴 등

[데미지 계산]
- 최종 피해 = max(1, 카드 기본 피해 + 강화 토큰 - 방호 토큰)
- 최소 1 피해 보장 (완전 무적 없음)

----------------------------------------------------------------
5. 이동 시스템 (체스 기물)
----------------------------------------------------------------
기물 이동 비용 (게이지):
- King:   상하좌우+대각 1칸.          비용 2 (0.5턴)
- Knight: L자 도약 (2+1칸). 중간 무시. 비용 2 (0.5턴)
- Rook:   상하좌우 직선 무제한.        비용 4 (1턴)
- Bishop: 대각선 직선 무제한.          비용 4 (1턴)
- Queen:  전방위 직선 무제한.          비용 6 (1.5턴)

이동은 공개카드 행동과 별개 선택지. 변주로 공격/방어에 이동 옵션 추가 가능.

----------------------------------------------------------------
6. 스킬 범위
----------------------------------------------------------------
- Linear(N): 상하좌우 N칸
- Diagonal(N): 대각 4방향 N칸
- All(N): 전방위 N칸
- Self: 자기 자신
방향 개념 없음. 모두 대칭 적용.

7. 장비 & 유품 시스템
----------------------------------------------------------------
[슬롯 구조]
- 인형당 무기/장갑/소품 각 1슬롯
- 유품은 3슬롯 중 반드시 1개 (이동 방식 결정)
- 기본 장비: 무기 없으면 주먹, 장갑 없으면 진흙 피부 자동 장착

[유품 (Relic)]
- 영혼이 깃든 장비. 체스 기물 이동 부여
- 장착 시 해당 영혼의 미니어처 모습으로 변신
- 이름 없는 영혼들. 플레이버 텍스트/일러스트로 암시
- 같은 이름 유품 = 같은 장비 기반, 다른 영혼. 덱 구성 동일 (이스터에그: 미묘하게 다른 버전 극소수)

[귀속 유품 (Bound Relic)]
- 분리 시 진흙 비용 발생
- 퍼마데스 시 일반 유품과 동일하게 회수 가능
- HP 연동 메커니즘 특화 (피흡, 저체력 각성 등)

[티어 네이밍]
- 무기/장갑: 티어1 농부의 / 티어2 도적의 / 티어3 용병의 / 티어4 기사의 / 티어5 영웅의
- 소품: 직업/신분 고유명 (영애의, 숲지기의, 성녀의 등)
- 레귤러/이레귤러 구조

[장갑 특성]
- 판금: 방호 높음. 이동 보조 거의 없음. 변주 6~7장
- 가죽: 방호 중간. 이동 보조 중간. 변주 4~5장
- 천: 방호 낮음. 이동 보조 많음. 드로우 특화. 변주 3~4장

[소품]
- 완전 자유 슬롯. 변주 0~10장
- 테마에 따라 무엇이든 가능
- 대부분 기본 효과 존재. 변주로 격변

[티어별 완성도]
- 저티어: 잡다한 변주. 애매한 공개카드. 답답한 싸움
- 고티어: 강력한 변주로 압축. 고유 유틸 공개카드. 완성형

----------------------------------------------------------------
8. 슬롯 레벨 & 성장
----------------------------------------------------------------
- 인형당 슬롯별 독립 레벨 (WeaponSlotLevel / ArmorSlotLevel / AccessorySlotLevel)
- 대표 레벨 = max(3슬롯). HP 기준
- 슬롯 레벨 = 해당 슬롯 장비 티어 상한
- 레벨업/다운그레이드: 진흙 투자/일부 반환
- 수복: 풀피 고정 비용. 대표 레벨 높을수록 비쌈

[진흙 재화 용도]
- 인형 생성 (퍼마데스 후. 슬롯 전체 Lv.1)
- 슬롯 레벨업/다운그레이드
- 인형 수복 (풀피 고정)
- 귀속 유품 분리
- 장비/유품 구매

[난이도]
- 이지/노말/하드 = 진흙 수급량만 다름. 전투/적 동일

----------------------------------------------------------------
9. 인형 운용
----------------------------------------------------------------
[6인 엔트리]
- 전투 투입 3마리 + 대기 3마리
- 퍼마데스: 슬롯 레벨 투자 진흙 + 일반 장비 소실. 유품은 회수

[덱 구성과 역할]
- 딜러형: 무기 7 / 장갑 3
- 탱커형: 무기 3 / 장갑 7
- 서포터형: 무기 3 / 장갑 3 / 소품 10
- 소품: 약점 보완 or 강점 극대화

[인형]
- 이름 붙이기 가능 (Golem.Name)
- 레벨 Lv.1~5

----------------------------------------------------------------
10. 전투 시스템
----------------------------------------------------------------
- 그리드 기반. 초반 5×5 → 중반 6×7 → 후반 8×8
- 투입 인형 최대 3마리. 전투 중 교체 불가
- 패배: 투입 인형 3마리 전멸
- HP 회복: 정비 화면에서 진흙 소모 (전투 후 자동 회복 없음)
- 최소 1 피해 보장 → 고레벨 인형 저렙 던전 투입 비효율

[전투 설계 의도]
- 3마리 협동 전술 지향 (RPG 원톱 아님)
- 브레이크 타겟 선택이 핵심 전술
  * 적 무기 브레이크 → 아군 생존
  * 적 장갑 브레이크 → 딜러 화력 극대화

[적 AI]
- 결정론적. 카드 우선순위 순서대로 CanUse 판단 후 사용
- 없으면 기본 이동
- 적 머리 위: 보유 변주 종류 표시

12. 게임 시작 & 튜토리얼
----------------------------------------------------------------
지급 장비:
- 농부의 장검 1개 (무기 유품. King 이동)
- 농부의 판금 갑옷 1개 (장갑 유품. King 이동)
- 성녀의 성서 1개 (소품 유품. King 이동)

시작 인형 구성:
- 인형 A: 농부의 장검 + 진흙 피부 + 없음 → 딜러
- 인형 B: 주먹 + 농부의 판금 + 없음 → 탱커
- 인형 C: 주먹 + 진흙 피부 + 성녀의 성서 → 서포터

전부 King 이동. 5×5 맵에서 시작.

13. 프로젝트 구조
----------------------------------------------------------------
솔루션 구조:
- GameCore: C# Class Library. 게임 로직. MonoGame 의존성 없음
- GameApp: MonoGame Project. 렌더링/입력/사운드. GameCore 참조

핵심 네임스페이스:
- GameCore.Domain.Units: Golem, Equipment, Relic, PublicCard, VariationCard, Deck
- GameCore.Domain.Units.Movement: MovementPattern, ChessPieceType
- GameCore.Domain.Battle: BattleState, BattleGrid, Tile, TileType
- GameCore.Domain.Battle.Gauge: ActionGauge
- GameCore.Domain.Map: WorldMap, DungeonNode, MapEdge
- GameCore.Domain.Characters: Necromancer, Enemy, Boss
- GameCore.Systems: BattleSystem, MapSystem, ProgressionSystem, ShopSystem
- GameCore.Scripting: IActiveSkill, IPassiveSkill, IDefaultSkill, SkillRegistry
- GameCore.Events: BattleEvent, MapEvent, GameEvent
- GameCore.Data: RelicData, EquipmentData, EnemyData, NodeData, CardData

핵심 도메인 모델:
- Golem: Id, Name, WeaponSlotLevel, ArmorSlotLevel, AccessorySlotLevel,
         Hp, MaxHp, WeaponSlot, ArmorSlot, AccessorySlot,
         ActionGauge, AccumulatedTurns, DefaultSkills[]
- Relic: Id, SoulName, Tier, SlotType, PieceType, IsBound,
         PublicCard, VariationCards[], GolemSprite
- PublicCard: CardId, Name, ActionType(Attack/Defense/Accessory),
              GaugeWeight, SkillClass, PassiveTokens[], RepairCost(N장),
              MoveOption, Illustration, Description
- VariationCard: CardId, Name, VariationType(Attack/Defense/Accessory),
                 IsIndependent, SkillClass, Illustration, Description
- IActiveSkill: CanUse(BattleContext), Execute(BattleContext)
- BattleContext: Caster, Targets[] (나머지는 BattleManager에 요청)
- RangeData: type(Linear/Diagonal/All/Self), range(int)
- ChessPieceType: Rook / Bishop / Knight / Queen / King
- TileType: Normal / Wall / Pit / EffectTile

데이터 관리:
- 정적 데이터 → JSON
- 스킬 로직 → C# 클래스 (리플렉션 런타임 바인딩)
- JSON의 skillClass → Type.GetType() → Activator.CreateInstance()
- 세이브 포맷 → JSON

로직-렌더링 분리:
- GameCore는 MonoGame 네임스페이스 참조 없음
- 입력 → Command 객체 → GameCore.Systems
- GameCore → GameEvent 발행 → GameApp 구독

14. MVP 범위
----------------------------------------------------------------
- 5×5~8×8 그리드 (장애물/낙사 없음)
- 유품 3~5개 (기물 종류별)
- 공개카드/변주 카드 10개 내외 (일러스트 없음, 텍스트)
- 적 2~3종
- 미니 던전 1개 (전투 5~7개 + 보스 1개)
- 토큰 3종 (내구도/강화/방호)
- 브레이크 시스템
